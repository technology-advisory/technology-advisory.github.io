Implementación y Gestión de Clústeres Hyper-V de Conmutación por Error

Un análisis exhaustivo sobre la arquitectura de redundancia de cómputo, almacenamiento iSCSI y la continuidad de servicios críticos.

1. Fundamentos de la Alta Disponibilidad (HA)

La virtualización moderna exige que los servicios no dependan de un único hardware físico. El **Failover Clustering** de Microsoft proporciona una infraestructura donde varios servidores (nodos) trabajan coordinados. Si un nodo experimenta un fallo de hardware, las máquinas virtuales se reinician automáticamente en los nodos restantes.

Análisis de Tolerancia a Fallos: No solo se protege contra fallos de CPU o RAM; un clúster bien diseñado protege contra fallos de red y de almacenamiento mediante rutas redundantes (MPIO).

2. Infraestructura de Almacenamiento Compartido

El pilar central de un clúster es que todos los nodos "vean" el mismo disco al mismo tiempo. En esta implementación, se utiliza el protocolo **iSCSI** para conectar los servidores a una cabina de almacenamiento externa (SAN/NAS).

El Rol Crítico del CSV (Cluster Shared Volumes)

A diferencia de los sistemas de archivos tradicionales donde solo un servidor puede escribir a la vez, el CSV permite que todos los nodos del clúster lean y escriban en el mismo LUN simultáneamente, permitiendo que las máquinas virtuales se muevan entre nodos sin perder la conexión con sus discos duros virtuales (VHDX).

3. El Mecanismo de Quórum y Testigo (Witness)

En un clúster, las decisiones se toman por mayoría de votos. Si un clúster tiene 2 nodos y pierden comunicación, ambos intentarían tomar el control, causando una corrupción de datos (Split-Brain). Para evitarlo, implementamos un **Testigo de Disco**.

Este testigo actúa como un "voto de desempate". El nodo que logre bloquear el acceso al testigo se mantiene activo, mientras que el otro se apaga de forma segura para proteger la integridad de la información.

4. Configuración de Redes de Clúster

Una arquitectura profesional separa los flujos de datos en redes lógicas aisladas:

  • Red de Gestión: Para la administración del sistema operativo y Active Directory.
  • Red de Heartbeat: Una línea de vida privada entre nodos para monitorizar el estado de salud.
  • Red de Live Migration: Dedicada al movimiento de la memoria RAM de las máquinas virtuales entre servidores.

5. Procedimiento de Despliegue mediante PowerShell

La validación es el paso más importante. Este proceso verifica que la latencia de red y los tiempos de reserva de disco son óptimos para la producción.

PS - Enterprise Deployment 
# Instalación de los componentes de Windows
Install-WindowsFeature -Name Hyper-V, Failover-Clustering -IncludeManagementTools

# Test de validación total (obligatorio para soporte de MS)
Test-Cluster -Node "NODE-01", "NODE-02"

# Creación del clúster con punto de acceso administrativo
New-Cluster -Name CLUS-INFRA -Node "NODE-01", "NODE-02" -StaticAddress 10.0.50.20

# Configuración del Quórum con disco dedicado
Set-ClusterQuorum -NodeAndDiskWitness "Cluster Disk 1"

6. Conclusiones y Mantenimiento

Con esta estructura, se pueden realizar mantenimientos en caliente. Gracias a la **Live Migration**, podemos mover las máquinas virtuales al Nodo B, actualizar el Nodo A, reiniciarlo y devolver las cargas de trabajo sin que los usuarios finales perciban una caída en el servicio.
